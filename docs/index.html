Booklit

Booklit is a static-site generator where the content is a first-class citizen.

Sections can be nested arbitrarily, either in-line or by including them from a separate file. Each section determines whether its sub-sections should be on a separate page, or inline. Header sizes will be set accordingly, so you're not spending your time adjusting <h1> and <h2>s.

Inter-section linking is done by referencing tags, rather than hardcoding URLs. When Booklit compiles your documents, it will raise an error if any references are broken. Tags can either point to an entire section, or a target element within a section, for example a definition for something in reference documentation.

The plugin system allows you to to define arbitrary \functions that you can call from your documents to generate content. This allows your content to become more semantically meaningful and easier to maintain.

Booklit separates presentation from content by having a separate rendering stage. With the HTML renderer, you can bring your own templates via Go's html/template package, and control exactly how everything should look.

Putting all of these things together, Booklit allows authors to write their content however they want to, tailoring the tool for your content, not the other way around. Interested? Let's get started.
