2 Getting Started

2.1 Hello, world!

First, create a file called hello.lit with the following content:

\title{Hello, world!}

I'm a Booklit document!

Then, run the following to build it in to ./docs/hello.html:

$ booklit -i hello.lit -o docs

2.2 Organizing with Sections

Next, let's try adding a section within our document:

\title{Hello, world!}

I'm a Booklit document!

\section{
  \title{Hi there!}

  I'm so organized!
}

2.3 Splitting Sections

To render each sub-section on its own page, simply call \split-sections somewhere in the section.

\title{Hello, world!}

\split-sections

I'm a Booklit document!

\section{
  \title{Hi there!}

  I'm so organized!
}

You may want to use this in combination with \table-of-contents, which generates a...table of contents:

\title{Hello, world!}

\split-sections

I'm a Booklit document!

\table-of-contents

\section{
  \title{Hi there!}

  I'm so organized!
}

2.4 References & Tagging

Having separate pages isn't very useful without a way to link to them. For this, use \reference:

\title{Hello, world!}

\split-sections

I'm a Booklit document! To read further, see \reference{hi-there}.

\section{
  \title{Hi there!}

  I'm so organized!
}

The first argument to \reference is the name of a tag to reference. At build time, references will resolve to their tag, and generate a link to it. By default, the name of the link is determined by the tag, so for a section it'll be the section's title. This can be overridden by passing a second argument to \reference:

\title{Hello, world!}

\split-sections

I'm a Booklit document! Consult \reference{hi-there}{this section}
for more information.

\section{
  \title{Hi there!}

  I'm so organized!
}
